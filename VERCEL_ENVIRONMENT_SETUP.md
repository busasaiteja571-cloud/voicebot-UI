# Vercel Environment Variables Setup Guide

## Problem Statement

When deploying an Angular application to Vercel, you may encounter this TypeScript compilation error:

```
TS2591: Cannot find name 'process'. Do you need to install type definitions for node? 
Try `npm i --save-dev @types/node` and then add 'node' to the types field in your tsconfig.
```

This occurs because Angular applications run in the browser, where `process.env` (a Node.js global) is not available at runtime. However, Vercel needs to inject environment variables during the build process.

## Solution Overview

This solution uses a **build-time environment injection** approach:

1. **At Build Time:** A Node.js script reads environment variables
2. **During Compilation:** These values are written into TypeScript source files as constants
3. **In Browser:** The compiled JavaScript contains hardcoded values, not `process.env` references

## What Was Done

### ✅ Step 1: Install Node.js Type Definitions

**Command executed:**
```bash
npm install --save-dev @types/node
```

**What this does:**
- Installs TypeScript type definitions for Node.js
- Allows TypeScript compiler to recognize `process` object during build

**Result:** `@types/node` now appears in `package.json` devDependencies

---

### ✅ Step 2: Update TypeScript Configuration

**File: `tsconfig.app.json`**

**Change made:**
```json
{
  "compilerOptions": {
    "outDir": "./out-tsc/app",
    "types": ["node"]  // ← Added 'node' here
  }
}
```

**What this does:**
- Tells TypeScript to include Node.js type definitions
- Enables `process` to be recognized during compilation
- This compilation happens only during build, not in the browser

---

### ✅ Step 3: Create Build-Time Environment Script

**File: `set-env.js`**

**How it works:**

```javascript
// Reads from environment at build time
const apiBaseUrl = process.env.API_BASE_URL || (isProduction ? '/api' : 'http://localhost:8080/api');

// Writes to TypeScript file BEFORE Angular compilation
const environmentFileContent = `export const environment = {
  production: ${isProduction},
  apiBaseUrl: '${apiBaseUrl}',
  geminiApiKey: '${sanitizeString(geminiApiKey)}'
};`;

// Writes the file
fs.writeFileSync(envFilePath, environmentFileContent, 'utf8');
```

**Key Features:**
- ✅ Runs only during build, not at runtime
- ✅ Escapes strings to prevent code injection
- ✅ Supports both production and development modes
- ✅ Provides sensible defaults if environment variables not set
- ✅ Includes colored console output for clarity

---

### ✅ Step 4: Update Package.json Scripts

**File: `package.json`**

**Changes made:**
```json
{
  "scripts": {
    "start": "node set-env.js development && ng serve",
    "prebuild": "node set-env.js production",
    "build": "ng build --configuration production",
    "build:dev": "node set-env.js development && ng build --configuration development",
    "watch": "ng build --watch --configuration development",
    "test": "ng test"
  }
}
```

**How it works:**

| Script | Purpose | Execution Flow |
|--------|---------|-----------------|
| `npm start` | Local development | `node set-env.js development` → `ng serve` |
| `npm run build` | Production build (Vercel) | `npm run prebuild` → `node set-env.js production` → `ng build --configuration production` |
| `npm run build:dev` | Development build | `node set-env.js development` → `ng build` |

**Why `prebuild`?**
- Automatically runs before `npm run build`
- No need to manually trigger it
- Standard npm convention

---

### ✅ Step 5: Update Environment Files

**Development: `src/environments/environment.ts`**
```typescript
export const environment = {
  production: false,
  apiBaseUrl: 'http://localhost:8080/api',
  geminiApiKey: ''
};
```

**Production: `src/environments/environment.prod.ts`**
```typescript
// This file will be auto-generated by set-env.js at build time
export const environment = {
  production: true,
  apiBaseUrl: '/api',
  geminiApiKey: ''
};
```

## Workflow Explanation

### Local Development (`npm start`)

```
npm start
  ├─ node set-env.js development
  │  └─ Reads: process.env.API_BASE_URL (if set)
  │  └─ Falls back to: http://localhost:8080/api
  │  └─ Writes to: src/environments/environment.ts
  │
  └─ ng serve
     └─ Compiles with environment.ts values
     └─ Starts dev server on http://localhost:4200
```

**Result:**
- Frontend connects to local backend at `http://localhost:8080/api`
- Environment variables not needed locally (defaults work)

### Production on Vercel (`npm run build`)

```
npm run build (on Vercel)
  ├─ npm run prebuild (automatically triggered)
  │  ├─ node set-env.js production
  │  │  ├─ Reads: process.env.API_BASE_URL (set in Vercel dashboard)
  │  │  │  Example: "https://voicebot-server-xyz.vercel.app/api"
  │  │  │
  │  │  ├─ Reads: process.env.GEMINI_API_KEY (set in Vercel dashboard)
  │  │  │
  │  │  └─ Writes to: src/environments/environment.prod.ts
  │  │     ```typescript
  │  │     export const environment = {
  │  │       production: true,
  │  │       apiBaseUrl: 'https://voicebot-server-xyz.vercel.app/api',
  │  │       geminiApiKey: 'your-key-here'
  │  │     };
  │  │     ```
  │  │
  │  └─ ng build --configuration production
  │     ├─ Compiles TypeScript
  │     ├─ Replaces environment.ts with environment.prod.ts
  │     ├─ The environment values are now hardcoded in JavaScript
  │     └─ Creates optimized build in dist/voicebot-ui/browser
  │
  └─ Vercel deploys the build
     └─ Browser runs JavaScript with hardcoded API URL
     └─ No need for process.env in browser ✓
```

**Result:**
- Frontend connects to production backend URL
- No TypeScript compilation errors ✓
- No browser errors about missing `process` ✓
- Environment variables injected safely at build time ✓

## Vercel Configuration

### Step 1: Set Environment Variables in Vercel Dashboard

**Go to:** Your Vercel Project → **Settings → Environment Variables**

**Add these variables:**

```
Key: API_BASE_URL
Value: https://your-backend-domain.com/api
Scopes: Production, Preview, Development (select all)

Key: GEMINI_API_KEY
Value: your_actual_gemini_api_key
Scopes: Production (select this scope only for security)
```

### Step 2: Verify Build Configuration

**In Vercel Dashboard:**

- **Build Command:** `npm run build` ✓ (This will automatically run `prebuild` → `set-env.js`)
- **Output Directory:** `dist/voicebot-ui/browser` ✓
- **Install Command:** `npm install` ✓

### Step 3: Deploy

Push to GitHub:
```bash
git add .
git commit -m "fix: Implement build-time environment injection for Vercel"
git push origin main
```

Vercel will automatically:
1. Install dependencies (`npm install`)
2. Build project (`npm run build`)
   - Which triggers `prebuild` → `node set-env.js production`
   - Injects environment variables into `environment.prod.ts`
   - Compiles with these hardcoded values
3. Deploy to Vercel's CDN

## Security Considerations

### ✅ Safe to Expose in Frontend

These values ARE safe to include in frontend JavaScript:
- ✅ `API_BASE_URL` - Just tells browser where to fetch from
- ✅ `GEMINI_API_KEY` - If restricted to frontend-only API calls

### ⚠️ Consider Backend Keys

If you have backend-specific API keys:
- ❌ DO NOT inject backend secrets into frontend
- ✅ DO keep backend secrets only in backend environment variables
- ✅ Backend receives requests from frontend at the public API endpoint

### String Sanitization

The `set-env.js` script automatically escapes quotes and backslashes to prevent code injection:

```javascript
function sanitizeString(str) {
  return String(str)
    .replace(/\\/g, '\\\\')  // Escape backslashes
    .replace(/'/g, "\\'");   // Escape quotes
}
```

**This prevents malicious values like:**
```
API_BASE_URL="'; console.log('hacked'); //"
```

## Troubleshooting

### Issue: "Cannot find name 'process'" Still Occurs

**Solution:**
1. Verify `@types/node` is installed:
   ```bash
   npm ls @types/node
   ```
2. Check `tsconfig.app.json` has `"types": ["node"]`
3. Delete `node_modules` and rebuild:
   ```bash
   rm -r node_modules
   npm install
   npm run build
   ```

### Issue: Environment Variables Not Injected

**Check:**
1. Vercel environment variables are set in dashboard
2. `prebuild` script runs: `npm run build --verbose`
3. Generated `environment.prod.ts` contains values:
   ```bash
   cat src/environments/environment.prod.ts
   ```

### Issue: Build Fails with SyntaxError in set-env.js

**Check:**
1. `set-env.js` is in project root: `ls -la set-env.js`
2. File has correct permissions: `chmod +x set-env.js`
3. No quotes/escaping issues in environment variables

### Issue: Different URLs for Different Environments

**Configure per environment in Vercel:**

In Vercel Dashboard → **Settings → Environment Variables**

- **Production:** `API_BASE_URL=https://api.example.com`
- **Preview:** `API_BASE_URL=https://preview-api.example.com`
- **Development:** `API_BASE_URL=https://dev-api.example.com`

Each preview/production deployment uses its scope.
